# Sitemap Utilities

This folder contains all sitemap-related utilities for generating the application sitemap.

## Structure

```
_sitemap/
├── types.ts           # Type definitions
├── utils.ts           # Helper functions for creating sitemap entries
├── static-routes.ts   # Static routes configuration
├── fetch-bonds.ts     # Fetch all bonds via API
├── fetch-content.ts   # Fetch blogs, news, and issuer notes via GraphQL
├── index.ts           # Centralized exports
└── README.md          # This file
```

## Files Overview

### `types.ts`
- Type definitions for sitemap entries
- Static route configuration interface

### `utils.ts`
- `getBaseUrl()` - Gets the base URL from environment variables
- `createSitemapEntry()` - Creates a single sitemap entry
- `createSitemapEntries()` - Creates multiple sitemap entries

### `static-routes.ts`
- `getStaticRoutes()` - Returns all static routes (home, pages, tools, etc.)
- Add new static routes here when creating new pages

### `fetch-bonds.ts`
- `fetchBondsForSitemap()` - Fetches all bonds via API
- Uses pagination to fetch all bonds
- Creates sitemap entries for `/bonds/detail/{isin}`

### `fetch-content.ts`
- `fetchBlogsForSitemap()` - Fetches all blog posts via GraphQL
- `fetchNewsForSitemap()` - Fetches all news posts via GraphQL
- `fetchIssuerNotesForSitemap()` - Fetches all issuer notes via GraphQL
- Uses pagination to fetch all content
- Creates sitemap entries for respective routes

## Adding New Routes

### Static Routes
Add new static routes to `static-routes.ts` in the `staticPaths` array:

```typescript
const staticPaths = [
  // ... existing routes
  "/your-new-page",
];
```

### Dynamic Routes
If you have new dynamic content (e.g., new content type), create a new fetch function in `fetch-content.ts`:

1. Create a GraphQL query
2. Create a fetch function similar to `fetchBlogsForSitemap()`
3. Export it from `index.ts`
4. Import and use it in `sitemap.ts`

Example:
```typescript
export async function fetchYourContentForSitemap(): Promise<SitemapEntry[]> {
  // Implementation similar to fetchBlogsForSitemap()
}
```

## Priority Guidelines

- **Homepage**: 1.0
- **High-value pages** (bonds, tools): 0.9
- **Content pages** (blogs, news): 0.8
- **Static pages** (about, contact): 0.8
- **Other pages**: 0.7

## Change Frequency Guidelines

- **Always**: Pages that change constantly
- **Hourly**: News, real-time data
- **Daily**: Blogs, frequently updated content
- **Weekly**: Bonds, issuer notes, most content
- **Monthly**: Static pages, tools
- **Yearly**: Rarely updated pages
- **Never**: Pages that don't change

## Notes

- The sitemap is automatically generated by Next.js at `/sitemap.xml`
- In sandbox mode, the sitemap returns an empty array
- All data fetching happens in parallel for better performance
- If any fetch fails, static routes are still returned as fallback
